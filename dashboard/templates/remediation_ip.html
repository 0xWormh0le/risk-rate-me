Dangerous IP addresses

These checks ensure that your IP address has not been identified as potentially malicious or dangerous.

If your IP is flagged positivley we provide the following data:

  • The last time the IP was flagged as dangerous (in Unix time or 0 if not listed recently)
  • The category of activity detected (Spyeware, malware, bot etc..)
  • The data source

Domain Name System Blackhole List (DNSBL) Checks

We currently check more than 150 blacklists. The blacklists we connect to are most commonly known as either a DNSBL (Domain Name System Blackhole List) or a RBL (Real-time Blackhole List). Most blacklists we check are geared towards blocking hosts involved with sending email spam, some are more specialized lists. These are a seperate category to the above checks and use seperate data feeds.

If your IP or Domain is flagged positivley we provide the following data:

  • The number of DNSBLs the host is listed on
  • the name of the DNSBL
  • the domain/hostname of the DNSBL
  • The data source text-record

What do I do if listed?

These test could indicate an already comprimised host/website. Immediately each item should be investigated. We recommend you take immediate steps to resolve these issues.

If you require additional assistance, we can provide a full in depth assessment that will help you identify and remediate any issues. This is an active assessment and is inherently invasive and intrusive. We require permission of the domain owner/controller to perform this assessment.
