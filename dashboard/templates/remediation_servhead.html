Server Header

If you are running a web server, that web server is probably showing the world what type of server it is, and possibly its version number. This information is ignored by most people, with the exception of hackers, who use this information to launch targeted attacks against your web server and version. In addition, if the version of your web server is known to be vulnerable to a specific exploit, the hacker would just need to use the exploit as part of his attack on your server.

Removing the entire header, while perfectly acceptable, is usually not necessary. However, it is recommended to remove the version numbers from the header. In the case that bugs exist for a specific web server version, including the version number can act as an invitation for script kiddies to try exploits against the server.

Masking the Server Header

Masking in Nginx

Edit /etc/nginx/nginx.conf and add this line (or remove comment from existinf line under http

server_tokens off;

Masking in Apache

Edit /etc/httpd/conf/httpd.conf and add the followng code at the ed of the config file:

ServerSignature Off

Enable on IIS

Masking in IIS

  1. Open the IIS Manager
  2. Select the website that Secret Server is running under.
  3. Select "HTTP Response Headers"
  4. Select the "X-Powered-By" HTTP Header and select "Remove"
